# 2026-04-22

- PrivaShield-Login-/Mandantenproblem analysiert: Nach dem Login wurde der aktive Mandant in `client/src/App.tsx` nur aus `localStorage` geladen, aber nicht automatisch anhand der erlaubten Mandanten initialisiert.
- Fix umgesetzt: `AppRoutes` lädt nun nach vorhandenem Token die Mandantenliste und setzt automatisch den ersten erlaubten Mandanten, falls keine gültige Auswahl vorhanden ist. Dadurch ist die Mandantenauswahl direkt nach dem Login nutzbar und Seiten benötigen keinen Umweg über das Dashboard mehr.
- Verifikation erfolgreich: `npm run check` und `JWT_SECRET=testsecret npm run build` liefen erfolgreich durch.
