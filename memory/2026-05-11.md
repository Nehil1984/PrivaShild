- Anschlussfix VulnLedger 0.3.1 umgesetzt und veröffentlicht:
  - Ursache für "nach Login kein Inhalt" lag frontendseitig im fehlerhaften Session-Ladevorgang: `loadSession()` wurde direkt während des Renderns über `if (loading && !user) { void loadSession() }` aufgerufen statt sauber in einem `useEffect`. Das war fragil und konnte den Nachladefluss nach Login stören.
  - Fix: Session-Initialisierung auf sauberen `useEffect` mit Cancel-Guard umgebaut.
  - Parallel dazu SQLite vollständig auf das neue Multi-Tenant-/Konzernmodell nachgezogen: users mit tenantIds, groups, customers mit groupId, assessments/findings/reports/evidence/retests inklusive tenant-/group-bezogener Felder.
  - `server/storage.ts` dispatcht jetzt echt zwischen lowdb und sqlite statt sqlite nur als Stub zu belassen.
  - Verifiziert wurden nun beide Pfade:
    - `npm run build:full`
    - `npm run lint`
    - HTTP-Login + Folge-Reads auf lowdb erfolgreich
    - HTTP-Login + Folge-Reads auf sqlite erfolgreich (`DB_BACKEND=sqlite`, Port 3001)
